第 0 阶段 · 立项：一段口述设定

项目始于淇淇口述的一段游戏设定：每个玩家出生是自由设定的变形金刚、等级由变形齿轮决定、三大陆三维度、八个神秘变形齿轮、从一级怪到最强 boss 的刷怪塔、像地核一样的地下之城……以及那条著名的世界规律——最软的是泥巴，最硬的是代码。

第一件事不是写代码，而是把口述整理成《设定集》：淇淇说过的每一个字原样保留，空缺处用「💡建议」标注，由淇淇逐条拍板。设定集从 v0.1 一路修订到 v1.1，成为整个项目唯一的权威依据——代码永远服从设定集。

关键决策

  • 技术路线：浏览器 3D 体素游戏（类 Minecraft），Three.js + Vite + 纯 JavaScript
  • 零外部资源：所有方块纹理、模型、特效全部用代码生成
  • 「挖一百格找宝箱」这类设定用真实地形 + 物理实现，不写特殊剧情脚本

第 1 章 · 初始城镇 MVP

从零手写体素引擎，做出第一个能玩的版本：Uint8Array 体素世界、chunk 面剔除网格、程序化纹理、AABB 物理、DDA 挖放；出生自定义机器人、宝座下一百格的矿石齿轮、打碎第三块诡异木板坠入的大地密室、刷怪塔前 20 层、标签制怪物、齿轮等级、8 步任务链、自动存档。验收方式：清档从零、按真实操作走完全部任务链。

印象深刻的 bug：俯视角最初限制在 -60°，玩家根本没法「垂直往下挖一百格」——而这恰恰是淇淇设定的核心玩法。教训：手感参数必须跟着设定走。

第 2 章 · 完整世界

淇淇拍板了全部空缺设定（八齿轮藏匿点、暗黑地狱金子开门规则、鲲鹏与鹏之药水、刷怪塔 1000 层与地狱魔龙），世界随之完整：

  • 384² 大世界 + 真实海洋：游泳、涉水十分钟机制、海底宫殿
  • 四个维度：主世界 / 暗黑地狱（岩浆湖、火海勇敢者试炼）/ 终极之地（符号回血怪、虚空）/ 千层塔竞技场
  • 八神秘齿轮全部就位：齿轮装备决定形态与技能（Q 地震 / C 喷火 / X 净化 / Z 隐身）
  • 收服大陆宠物（打败即收服、可骑乘）、作者小岛商店、背包四页签、五级剑甲合成（代码最硬）、昼夜与床、船、三档视角
  • 大结局：集齐八齿轮回驱动核心旁的作者神殿

V3 · 品质大版本（正式更名「机甲时代」）

淇淇实测后提出十项质量问题，整版重做品质。七个里程碑各自独立提交验收：

里程碑内容
M1 修复层船驾驶（浅水探测差一格的 bug + A/D 转向 + 惯性）、鲲鹏巡飞与俯冲 AI、禁地守卫巡逻、陆地怪避水、水流扩散 + 海面动画
M2 画质层抗锯齿、体素顶点 AO、mipmap、完整天空系统（日月星云 + 光照随时间）、噪声纹理重绘、渲染距离剔除
M3 世界层512² 地图重布局（岛间 141~177 格开阔海面）、存档 v3 与链式迁移
M4 模型层机器人 v2 五形态精修、怪物全员肢体动画（扇翅/触手/蛛腿/蟹钳）、鲲鹏三节可折巨翼
M5 特效层对象池粒子系统、四技能专属特效、挖掘碎屑、死亡烟花、相机震动
M6 UI 层机甲主题重皮（钢灰+青金+切角面板）、世界 boss 血条、地标指南针

印象深刻的 bug：禁地守卫纹丝不动，查了三层才找到真凶——禁地岩体建在地形上方但没更新高度表，守卫出生时被整个埋进了石头里，物理碰撞每帧把它的速度清零。教训：「怪物不动」不一定是 AI 的锅。

V3.1 · 体验反馈

  1. 水边自动登岸（在水里再也不会困住）
  2. 右上角小地图（海岸线/高度明暗/地标/朝向箭头）
  3. 代码矿石字符环绕特效（设定原文「字母连成线」的动态化）
  4. 渐进流动水：浅水漫过来 → 1.2 秒涨满整格；静水面微降更像液体
  5. 挖掘破碎动画：四阶段裂纹蔓延 + 末段抖动 + 分阶碎屑

V4.0 · 初始城镇六区大重建

淇淇手绘了一张新地图，把出生地从「一座塔 + 一堆树」升级成六个区域的完整大岛。第二章之后最大的一次世界重构，一次做完、一次发布。

区域内容
① 中央城市城墙 + 四门 + 十字大道 + 作者之塔，22 座可进入房屋、市集、16 居民 + 4 类齿轮商店。城内绝对安全，怪物进不来
② 竹林（西）密竹 + 大熊猫 + 三层竹林神殿（竹躯干熊猫头图腾 / 远古熊猫 Boss / 宝藏），隐藏图腾宝箱
③ 巨石阵（西北）内外双圈石柱，中央下挖一百格 → 三块诡异木板 + 大地齿轮（第一章主线搬家）
④ 矿石群山（北）五座主峰，确定性矿脉：普通矿/金子多，钻石中，红蓝宝石/代码少，图腾节点全山 1~3 个
⑤ 鬼城遗址（东）约 20 处焦黑废墟 + 局部烟雾/火星/撕裂声，变异坦克/装甲车 + 红色邪恶巨龙（火焰光柱）
⑥ 森林（南）密林 + 35 格巨型古树 + 两层森林神殿（吊灯/花瓶/地毯/小人图壁画 / 红眼睛激光武器）

两件新神器：神秘图腾（伤害 ×2）与红眼睛（100 伤害即时激光）。刷怪塔迁到西侧小岛（石桥连接），地狱门迁到东南海岸，地下之城入口改到城内广场。

工程做法：六区拆成 worldgen/island/ 独立模块，新增 zones.js 统一区域系统（一个 zoneAt(x,z) 纯函数驱动安全区/刷怪/指南针/区域标题/鬼城雾）。写了 scripts/check-world.mjs——node 直跑世界生成，100 条断言把关键坐标钉死，每次改地形先跑它。
这次防的坑：项目在「坐标漂移」上栽过两次（V3 平移、V3.6.1 地下城）。这次靠 check 脚本 +「模块内禁止裸数字地标、一律引 POS」两条铁律避免复发，顺手修掉三处遗留旧坐标。老档自动升级 v4——等级/齿轮/任务/装备/宠物/其他大陆建造全保留，只清旧主岛的方块改动。

工程方法论

  1. 设定即法律：淇淇写的内容一字不改；工程只能在空缺处提建议并等拍板
  2. 每个里程碑独立验收：浏览器内真实操作或模拟时间快进，全部通过才提交
  3. 调试基建先行：debug 模式 + 假时间戳快进，让「挖一百格」秒级验证
  4. bug 找病根不贴膏药：每个修复都先定位到行号
  5. 对小朋友友好是设计约束：摔不死、死亡不掉落、被困按 H 回城、触控板全键盘可玩
