# 机甲时代（Age of Mechs）· 开发文档

> 从一段口述设定到一个完整的开源体素世界 —— 立项到现在的全部开发历程
> **原案 / 创世神**：淇淇（9 岁） ｜ **工程实现**：Claude（Anthropic） ｜ **仓库**：github.com/zdywrnm/Age-of-Mechs

## 第 0 阶段 · 立项：一段口述设定（2026-07-21）

项目始于淇淇口述的一段游戏设定：每个玩家出生是自由设定的变形金刚、等级由变形齿轮决定、三大陆三维度、八个神秘变形齿轮、从一级怪到最强 boss 的刷怪塔、像地核一样的地下之城……以及那条著名的世界规律——**最软的是泥巴，最硬的是代码**。

第一件事不是写代码，而是把口述整理成《设定集》：淇淇说过的每一个字原样保留，空缺处用「💡建议」标注，由淇淇逐条拍板。设定集从 v0.1 一路修订到 v1.1，成为整个项目唯一的权威依据——**代码永远服从设定集**。

**关键决策**：
- 技术路线：浏览器 3D 体素游戏（类 Minecraft），Three.js + Vite + 纯 JavaScript
- 零外部资源：所有方块纹理、模型、特效全部用代码生成
- 「挖一百格找宝箱」这类设定用真实地形+物理实现，不写特殊剧情脚本

## 第 1 章 · 初始城镇 MVP（2026-07-21 ~ 07-22）

从零手写体素引擎，做出第一个能玩的版本：

- **引擎**：Uint8Array 体素世界、chunk 面剔除网格、程序化纹理图集、AABB 物理、DDA 射线挖放
- **内容**：出生自定义机器人、作者之塔（宝座下一百格的矿石齿轮）、巨石阵（打碎第三块诡异木板坠入密室）、刷怪塔前 20 层、标签制怪物、齿轮=等级、T 变形、8 步任务链、localStorage 存档
- **验收方式**：清档从零、按真实操作走完全部任务链

**印象深刻的 bug**：俯视角最初限制在 -60°，玩家根本没法「垂直往下挖一百格」——而这恰恰是淇淇设定的核心玩法。教训：**手感参数必须跟着设定走**。

## 第 2 章 · 完整世界（2026-07-22）

淇淇拍板了全部空缺设定（八齿轮藏匿点、暗黑地狱金子开门规则、鲲鹏与鹏之药水、刷怪塔 1000 层与地狱魔龙），世界随之完整：

- **384² 大世界 + 真实海洋**：游泳、涉水十分钟机制、海底宫殿
- **四个维度**：主世界 / 暗黑地狱（岩浆湖、火海勇敢者试炼、恶魔恶龙）/ 终极之地（符号回血怪、虚空）/ 千层塔竞技场
- **八神秘齿轮全部就位**：齿轮装备决定形态与技能（Q 地震 / C 喷火 / X 净化 / Z 隐身）
- **系统**：收服大陆宠物（打败即收服、可骑乘）、作者小岛商店、背包四页签、五级剑甲合成（代码最硬）、昼夜与床、船、三档视角、鲲鹏 boss 掉药水与翅膀
- **大结局**：集齐八齿轮回驱动核心旁的作者神殿

## V3.3 · Blender 机甲建模（2026-07-23）

淇淇觉得盒子拼的机器人「有点糙」，于是**用 Blender 脚本化建了一台低多边形硬表面机甲**（倒角、发光胸核、关节件、背包），导出 GLB 接进游戏。关键是没丢原有功能：四部位换色按材质名克隆、走路动画按关节节点绑定、头型/眼睛按配置显隐——加载失败还会回退到盒子模型兜底。

**踩坑**：Blender 立方体缩放系数应等于目标尺寸（第一版全炸开）、合并零件前要先应用倒角修改器、中文界面节点得按类型查找。

## V3.6 · 远程炮（2026-07-23）

给机器人加**远程武器**（G 键 / 触屏🔫，按住连发）：弹药随收集的元素齿轮升级五档——能量弹 → 光明射线（穿透）→ 三连水弹（散射）→ 烈焰爆弹（爆炸溅射）→ 神秘追踪弹（自动咬向最近怪物）。从此不用贴脸也能战斗。

**踩坑**：追踪弹一开始绕着怪转圈打不中（转向慢+速度快形成轨道），加「球形近炸引信」+提高转向率解决。

## V3.6.1 · 修复地下之城（2026-07-23）

淇淇发现「地下城的驱动核心、光明齿轮都找不到，路也不通」。查出是**同一个坐标错位 bug**：512 大地图重布局时地下城空腔挪到了城镇正下方（96–144），但驱动核心、光明齿轮箱、神殿③、入口楼梯的坐标还停留在旧位置（152–180）——全被生成在空腔外的实心石头里。修正坐标 + 把断裂坡道换成竖井（按 E 双向传送 + 指南针地标）。教训：**大改布局时，凡引用坐标常量的结构都要逐个核对是否还落在范围内**。

## V4.0 · 初始城镇六区大重建（2026-07-24）

淇淇手绘了一张初始城镇岛的新地图，把出生地从「一座塔 + 一堆树」升级成**六个区域的完整大岛**。这是第二章之后最大的一次世界重构，一次性做完、一次发布。

**六个区域**（按淇淇的手绘布局）：
1. **中央城市**——城墙围起来的机甲古城，四个大门、十字大道、作者之塔居中、22 座可进入房屋、市集摊位、16 名流动居民 + 4 类齿轮商店（食品/建材/战斗/交通）。城内是**绝对安全区**：怪物不生成、不追进来、连子弹都进不来。
2. **竹林（西）**——8~14 格高的密竹、野生大熊猫、三层竹林神殿（一层竹躯干熊猫头大图腾、二层封闭 Boss 场住着**远古熊猫**、三层宝藏）。竹林深处藏着**神秘图腾宝箱**。
3. **巨石阵（西北高地）**——内外双圈石柱，中央往下挖一百格是三块诡异木板 + 大地齿轮箱（第一章主线原样保留，只是搬了家）。
4. **矿石群山（北）**——五座主峰，山体里塞满确定性矿脉：普通矿/金子多，钻石中等，红蓝宝石/代码稀少，**神秘图腾节点**全山只有 1~3 个。
5. **鬼城遗址（东）**——约 20 处焦黑废墟、局部烟雾与火星、撕裂的环境音，出没**变异坦克 / 变异装甲车**和地狱鬼怪，天上盘旋一条**红色邪恶巨龙**（会放火焰光柱）。
6. **森林（南）**——高密林、一棵 35 格高的巨型古树、两层森林神殿（一层吊灯/花瓶/地毯/小人图壁画，二层大箱开出**红眼睛**激光武器）。

**两件新神器**：
- **神秘图腾**——灰身躯、红边框、两双红眼；装备后玩家所有主动伤害 ×2（近战/机甲炮/激光/技能全吃）。唯一，重复获得转 25 齿轮。
- **红眼睛**——即时红色激光，射程 40 格、固定 100 伤害（图腾加成后 200），装上后 G/🔫 从机甲炮变激光，卸下自动恢复。

**工程做法**：这次刻意把六区拆成 `src/worldgen/island/` 下的独立模块（town/bamboo/henge/mountains/ghost/forest），不再往一个 structures.js 里堆；新增统一的**区域系统** `zones.js`（一个 `zoneAt(x,z)` 纯函数驱动安全区、刷怪、指南针、区域标题、鬼城雾效）。还写了 `scripts/check-world.mjs`——node 直接跑世界生成，用 100 条断言把所有关键坐标钉死（出生点/三条竖井主线/四门大道/石桥连通/矿脉计数/各区宝箱），每次改地形先跑它。

**踩坑**：这个项目在「坐标漂移」上栽过两次（V3 平移、V3.6.1 地下城），这次靠 check 脚本 + 「模块内禁止裸数字地标、一律引 POS」两条铁律避免复发，顺手修掉了三处遗留的旧坐标（terrain 植树用旧岛心、调试传送点旧塔坐标）。

**存档兼容**：老档（v3）首次载入自动升到 v4——等级、齿轮、任务、装备、宠物、八大齿轮、塔进度、其他大陆/维度的建造全部保留，只清理旧主岛范围内的方块改动，位置回到城南出生点。

## V5.0 · 世界大重建：大幅扩大 + 有机化（2026-07-24）

V4 六区发布后，淇淇自己先扩了一版（扩岛 + 平原郊区 + 零散 NPC），但**仍然很紧凑**，并提了三条硬要求：世界要**大幅扩大**（性能不是顾虑，以后还要加更多同样大的岛）、各区/城市/郊区都要**扩大加细节**、区域要做成**不规则有机形状、区与区之间自然过渡**（不再是一块一块的方形），地形与分区都要**仿真自然**。外加一个 bug：**居民和怪物的脸朝移动的反方向**。

这是「世界重建」级的一次工程，拆成 11 段提交、每段都跑 `check:world` + 浏览器实测：

**① 世界扩容 + 岛屿重定位**：主世界 512³ → **1024×200×1024**（面积 4×）；初始岛心从 (128,128) 挪到 **(320,320)**、半径 116→**210**，远海地标全部等比外扩。1024² 扁平数组约 200MB，桌面无碍——世界尺寸做成**单一常量**，真机若 OOM 一行改回 768 即可。

**② 消灭坐标漂移的根**：这个项目在坐标漂移上栽过三次（V3 平移、V3.6.1 地下城、V4 三处遗留）。这次一劳永逸——在 config.js 顶部立**唯一基准点** `IC={x,z}` + `isl(dx,dz)` 派生函数，所有初始岛坐标全部改成相对岛心派生。**以后挪岛、加新岛只改 IC 一个数**。地下城空腔也顺手改成「腔心 = 岛心 = 城心」，根治历史偏心坑。

**③ 有机区域系统（域扭曲 Voronoi）**：区域方形拼接的病根是 `zoneAt` 纯矩形/圆判定。重写为**站点场**——先用 fbm 噪声**扭曲**采样坐标，再算到各区站点的归一化距离，最近且在辖域内者归该区、否则是平原过渡带。于是区界蜿蜒有机、区与区之间自然留出平原带。地形与分区**共享同一个 warp 实例和站点表**（单一真源），这是「地形跟着区域走、结构不落错地形」的机制根基。

**④ 地形仿真**：硬圆盘海岸线改成 `半径 − fbm×46` 的**蜿蜒海岸**（海角海湾摆动）；每区高度用区域权重（smoothstep）**软混合**——矿山拔起真正的山体、森林起伏丘陵、鬼城下陷裂地、竹林缓丘、巨石阵台地，区界平滑归零无台阶。

**⑤ 细节铺装**：新增 `decor.js`（路灯/水井/喷泉/花丛/篱笆/篝火/池塘/行道树）+ 6 种装饰方块（花/草/栅栏/砾石/篝火）；六区改用**密度场铺装**（`scatter` + 区域权重，核心密、边缘疏，面积翻倍数量自动跟上）；中央城市扩到 96×96、程序化生成约 32 座房 + 十字大道路灯行道树 + 市集喷泉；环城平原带撒农舍/农田/池塘 + 四门放射土路，让「城市 → 郊区农田 → 六区」自然过渡。

**⑥ 朝向 bug**：模型面朝 −Z，`atan2(-dx,-dz)` 本就朝移动方向，原代码多了一次 `+Math.PI` 二次翻转。修了居民、怪物追击、怪物游荡三处——**QA 阶段又揪出第四处**：宠物（本质是被收服的怪物、共用同一套模型）的 `pets.js` 也带着 `+Math.PI`，一并删掉。

**存档兼容**：坐标系整体变了，老档（v4/v4.1）载入自动升到 rev 4——**清空主世界方块编辑/船/位置**回新出生点，但等级、齿轮、八大齿轮、装备、宠物、塔进度、任务、宝箱、地狱与终极之地的建造**全部保留**。node 桩测 17 项 + 浏览器实测（19 级/90 齿轮/8 齿轮/塔 88 全保留、玩家从旧坐标重置到新出生点）双验证。

**踩坑**：1024 世界一开始把全部 4096 个 chunk 都建出来，FPS 从 60 掉到 20。加了**按玩家位置增量流式建 chunk**（`streamAround`）+ 收一点渲染距离用雾遮边界，只建 ~556 个，FPS 回到 60。教训：**世界一放大，「一次建满」就不再免费**。

## 工程方法论（给未来的自己）

1. **设定即法律**：淇淇写的内容一字不改；工程只能在空缺处提建议并等拍板
2. **每个里程碑独立验收**：浏览器内真实操作或模拟时间快进，全部通过才提交
3. **调试基建先行**：`?debug=1` 暴露内部对象 + 假时间戳快进，让「挖一百格」这类流程能秒级验证
4. **bug 找病根不贴膏药**：船开不动是采样差一格、守卫不动是被埋进结构——每个修复都先定位到行号
5. **对小朋友友好是设计约束**：摔不死、死亡不掉落、被困按 H 回城、触控板全键盘可玩
6. **坐标必须单一真源**：栽过三次坐标漂移后立铁律——全岛坐标由一个基准点 `IC` 派生，挪岛只改一个数；地形与分区共享同一噪声实例，杜绝「结构落错地形」
